
总的来说，我们要开发一个ACP也就是泛用的 应用控制协议，其中主要包括两个组件：
- 语言模型大脑模块：用来管理并执行具体任务。用户意图识别、任务拆解、执行规划、状态管理。通过识别用户需求，来拆解达成任务的具体方案和流程。
    - 比如：用户要求将 某个群中今天大家讨论的关于外星人飞船的内容整理成文章，用非AI的文笔，另外打开浏览器，找几张最近的外星人UFO的图片，并判断一定要相关的，然后进入到小红书发送相关文章。
- UI操作的眼睛和手：这是一个解耦并且独立的模块，主要是两个功能
    1. 感知屏幕，输出结构化的内容给LLM进行可操作性的参考。
    2. 操作屏幕，基于LLM输入的操作指令，自动匹配相关交互控件，并执行正确的比如点击、滑动、返回、home按钮、多窗口按钮等动作。




# 这是我跟Gemini讨论的一个需求

以下是根据我们的讨论，为您整理的 **APP 自动化控制 Agent (ACP-Grounding) 产品需求文档 (PRD)**：

### **项目名称**：轻量化 APP 自动化控制 Agent 框架

**1. 产品愿景**
构建一个低延迟、可快速落地且易于后期维护升级的 APP 自动化执行框架。通过解耦感知与决策，实现对通用 APP 的跨平台自主操作。

**2. 核心架构设计**
*   **解耦化感知层**：利用视觉模型（如 OmniParser 或 Grounding DINO + SAM）将屏幕截图解析为结构化的 JSON 数据（包含组件 ID、类型、位置及语义描述），解决 HTML 树不稳定的问题。
*   **轻量化决策层**：选用小参数量模型（如 **Qwen2-VL-2B** 或 **ShowUI-2B**）作为大脑，接收结构化数据并进行高层路径规划。
*   **交互执行层（UI Grounding）**：将模型指令精准映射为像素级操作。初期采用坐标生成模式，后期可升级为基于动作注意力头的“无坐标定位”模式。

**3. 关键功能需求**
*   **自定义 MCP Server**：自研基于 **Model Context Protocol (MCP)** 的服务端。核心职责是封装视觉解析工具，向决策模型提供标准化“可交互选项”列表，并执行底层的 Grounding 点击操作。
*   **页面转换图 (PTG) 管理**：在管理层引入 PTG 维护页面状态，确保多页面交互时逻辑的一致性，方便后期手动干预和逻辑修正。
*   **闭环反馈机制**：系统需捕捉操作后的页面跳转状态，若未达到预期，需将错误反馈给决策层进行迭代修正。

**4. 维护与升级策略**
*   **插拔式设计**：通过 MCP 协议标准化接口，后期可无缝更换更高分辨率的视觉模型或更强的逻辑模型，而无需重写 APP 控制逻辑。
*   **性能演进**：支持从“模块化”向“端到端 VLA”架构过渡；后期可引入**接地验证器 (Verifier)** 减少误触，提高执行成功率。

**5. 验收标准**
*   成功率：PTG 跳转覆盖率 (PCR) 需达到 90% 以上。
*   响应速度：在端侧设备上，单步决策延迟控制在秒级。

移动端AI大爆发：改变我们理解UI界面的5个颠覆性发现

1. 引言：当我们试图让AI“看懂”手机屏幕时，我们在难为它吗？

在移动互联网时代，存在一个普遍的认知误区：认为像 GPT-4V 这样的通用多模态大模型（MLLMs）可以像理解自然摄影图像一样，完美处理所有 UI 界面。然而，作为 UI/UX 架构师，我们深知移动端屏幕并非普通的“图像”，而是一个高度密集的表层交互层（Surface-level Interaction Layer）。

当你将一张长宽比极端的手机截图交给通用模型，并要求它精确定位一个仅占面积不到 0.1% 的微小图标时，模型往往会陷入“视而不见”的窘境。这是因为通用模型难以处理 UI 界面特有的矢量级密度与光栅化表现之间的矛盾。近期，随着 Ferret-UI、SAM 3 以及 SeeClick 等研究的突破，我们正在见证 AI 如何从“看像素”进化为“懂逻辑”，这场技术革命正彻底重塑我们对移动交互的理解。


--------------------------------------------------------------------------------


2. 突破“分辨率壁垒”：为什么“Anyres”是移动端AI的杀手锏

传统多模态模型通常会将输入图像强制缩放为正方形，这对于具有狭长比例的手机屏幕是致命的。

根据 Apple 的最新研究《Ferret-UI: Grounded Mobile UI Understanding》，直接调整大小会导致关键视觉信号的丢失。为了保护这些信号，研究者引入了 “Any-resolution (anyres)” 技术。其核心逻辑在于动态网格划分：

* 1x2 与 2x1 划分逻辑： 模型会根据原始屏幕的长宽比自动选择切分方式。Portrait（纵向） 屏幕被水平划分为两个子图，而 Landscape（横向） 屏幕则进行纵向切分。
* 保护交互靶点： 这种技术不仅放大了细节，更重要的是防止了缩放带来的比例失调。在移动端 UX 中，依赖长宽比的点击靶点（Aspect-ratio-dependent tap targets） 是交互的核心。Anyres 确保 AI 捕捉到的不仅是图标，更是这些图标在交互空间中的精确位置。

“依靠直接调整大小的低分辨率图像会导致关键视觉信号丢失，这些信号对于屏幕理解和交互至关重要。” —— Ferret-UI


--------------------------------------------------------------------------------


3. SAM 3 进化：从“点选”到“语义搜索”的质变

Meta 在 2025 年 11 月发布的 Segment Anything 3 (SAM 3) 为 UI 自动化带来了底层能力的质变。它不仅仅是一个分割工具，更是一个具备**提示词组合分割（Promptable Composable Segmentation, PCS）**能力的视觉中枢。

* 跨模态语义映射： 开发者现在可以输入“购物车图标”或“正在加载的进度条”，SAM 3 就能在静态图像甚至是动态视频（这对于理解交互动效至关重要）中生成精确的掩码（Mask）。
* 掩码解码器（Mask Decoder）的架构深度： SAM 3 的解码器采用了轻量级两层 Transformer。其核心在于图像嵌入（Image Embedding）与提示词 Token 之间的**多层交叉注意力（Multi-layer Cross-attention）**机制。这使得模型能将高层的文字描述直接映射到像素级的坐标空间。
* UI 标注的生产力革命： 借助这类能力，如 Label Assist 这样的工具可以实现自动化 UI 标注，通过语义理解大幅提升训练专用模型的数据效率。


--------------------------------------------------------------------------------


4. 术业有专攻：为什么专用模型在 UI 底层任务上击败了 GPT-4V

尽管 GPT-4V 是多模态领域的标杆，但在精细化的 GUI Grounding（界面定位）任务中，它正面临专用模型的强力挑战。由于 GPT-4V 依赖的 Set-of-Mark (SoM) 标签有时会遮挡极小的 UI 组件，导致其在密集布局下的表现受限。

根据 SeeClick 在 ScreenSpot 基准测试中的数据对比，专用模型在“点击预测”上的统治力令人震惊：

任务类型	GPT-4V (SoM 辅助)	SeeClick / Ferret-UI	专用模型的底层优势
Android 图标识别	4.8% (Grounding Acc)	52.0% (Grounding Acc)	SeeClick 通过 1M 规模的 GUI 样本进行专项预训练
文字识别 (OCR)	良好	优秀	Anyres 减少了小字符的像素粘连
GUI Grounding (平均)	16.2%	53.4%	针对“点击点”预测而非框选，更符合人类操作逻辑
复杂窗口分类	局限性明显	适应性强	专用模型具备更深层的 UI 组件先验知识

这种 10 倍以上的性能差距表明，理解 Visual Affordances（视觉可见性/意向性） 需要的不仅仅是大参数量，更需要对 UI 交互逻辑的深耕。


--------------------------------------------------------------------------------


5. 建立“语义词典”：AI 如何学会人类的 UI 逻辑？

要让 AI 实现功能推理（Function Inference）——即推测出“这是一个播客订阅页面”而非仅仅是“一个列表”——必须建立符合人类认知的设计语义系统。

Ranjitha Kumar 等研究者通过对 Rico 数据集（包含 9.7k 个应用）中的 73k 个 UI 元素进行迭代编码，构建了一套庞大的语义词典：

* 结构与功能解耦： 词典包含 25 种组件（如 Drawer、Modal）和 197 个文本按钮概念。
* 功能的深层语义： AI 能够理解“国旗（national_flag）”图标不仅是地理位置，还代表语言切换。这就是“功能语义（Functional Semantics）”的力量。
* 异常检测机制： 不同于 Ferret-UI 使用 Visual Sampler 处理区域特征，这项研究巧妙地结合了 CNN 与高斯混合模型（Gaussian Mixture Model）。通过分析 CNN 最后一层的 Softmax 激活向量，模型能以 94% 的准确率识别 135 种图标类别，并能准确识别出不属于任何已知类别的“视觉异常”。


--------------------------------------------------------------------------------


6. 结语：迈向真正的“视觉智能体”时代

从 SAM 3 实现的像素级动态分割，到 Ferret-UI 对长宽比的精准适配，再到 Rico 数据集构建的深层语义词典，AI 正在从一个“屏幕朗读器”进化为真正的视觉智能体（Visual Agents）。

这种范式转移将带来两个层面的变革：

1. 辅助功能的重塑： 利用 Ferret-UI 等模型的 Screen Recognition 启发式算法，我们可以为那些开发者疏于维护的 App 实时生成精准的元数据（Metadata），为视障用户开启一扇窗。
2. 交互路径的压缩： 当 AI 能比人类更快、更准地理解屏幕上的每一个交互意图时，我们可能正处于“手动点击”时代的终结点。
